面向边缘机器人运行时的编译引导内存布局

摘要

这条工作线连接了编译基础设施与具身部署问题，重点在于让模型之外的系统部分同样具备可分析性。

年份 2025
发表场合 Preprint Draft
我的角色 编译管线、运行时设计

Zhang, Y. Compiler-Guided Memory Layouts for Edge Robotics Runtime. Preprint Draft, 2025.

关键词
CompilerRuntimeEdge Robotics

01

A systems bridge

我一直对“模型之外的那部分系统”更感兴趣：数据如何排布、缓存如何失配、部署时哪些延迟其实与推理本身无关。

这项工作就是想把这些原本零散的问题重新纳入编译和运行时的共同视野。

02

Compiler-facing decisions

设计重点是让静态分析结果能够落到可执行的布局策略上，而不是停留在离线报告里。为此，内存块的组织、生命周期标注和调度提示都需要一套统一表达。

如果这套表达足够稳定，它未来也能迁移到其他端侧系统，而不只服务单一机器人栈。

03

Publication value

这类工作很适合 publication 页面，因为它需要同时展示问题、架构图、实验轨迹和尚未收束的部分。

我会把它保留成一条中间态的示范条目，代表那些仍在演化中的研究框架。
