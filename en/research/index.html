Research note

Embodied intelligence and VLA systems, studied as longer operational structures.

This line is less about isolated model scores and more about how perception, reasoning, execution, and memory can be organized into an embodied system that keeps evolving.

“The embodied systems worth pursuing are not the ones that only demo well once. They keep structure, memory, and recovery alive across longer task chains.”

Task planning before spectacle

Memory that survives long horizons

Evaluation under sparse supervision

Systems that remain editable

Research note

Compiler, runtime, and database systems studied under interpretability and constraint.

This direction brings database execution paths, compiler infrastructure, and deployment runtime behavior into the same frame to ask how complex systems remain understandable under pressure.

“What keeps me here is not only the performance result. It is the question of why a system still holds under certain constraints, and how that structure can be split, moved, and reused again.”

Execution paths over isolated optimizations

Compiler structure before tooling sprawl

Runtime behavior under deployment pressure

Benchmarks that explain, not just rank

Research note

Reverse analysis and runtime security, organized as reusable methodology.

This direction is more method-oriented: how can call chains, memory access patterns, and runtime observations be translated into stable audit structures rather than isolated intuition?

“What matters in security analysis is rarely one lucky hit. It is whether you can explain why you noticed it, and turn that attention into a method someone else can reuse.”

Signals before slogans

Runtime traces over intuition alone

Pattern libraries for audit work

Methodology that survives new samples
